Software Applications and Operation Systems Checklist

Maintained by Jessie Li

Click the test case name to see the test case description:

Checklist
Description
Test Case Name
(a)Executing Function from Keyboard

 Actions can be operable only be keyboard.

 keyboard_access

(b)Accessibility Features

 Accessibility features built into the mozilla can typically be turned on or off by a user.

 build_in features

(c)Input Focus

 Provide a visual focus indicator that moves among interactive objects as the input focus changes.

 focus indicator

(d)User Interface Element

 Sufficient information about a user interface element shall be available to assistive technology.

 ui element

(e)Bitmap Images

 If an image is used to identify programmatic elements, the meaning of the image must be consistent throughout the application

 bitmap image

(f)Textual Information

 Textual information shall be provided through operation system functions for displaying text.

 text_info

(g)User Selected Attributes

 Applications shall not override user selected contrast and color selections and other individual display attributes.

 user_attributes

(h)Animation

 Provide an option to display animation in a non-animated presentation mode.

 animation

(i)color Coding

 Use color as an enhancement, not as the only way to convey information or indicate an action

 color_coding

(j)Color and Contrast Settings

 When color customization is supported, provide a variety of color selections capable of producing a range of contrast levels

 color_contrast

(k)Flashing or Blinking Text

 Avoid the use of blinking text, objects and other elements. flash_blink

(l)Electronic Forms

 The form allow people using AT to access the  information, field elements and functionality required for completion and submission of the fform, includingall directions and cues.

 eletro_form
